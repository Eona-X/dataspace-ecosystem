# Kafka Proxy Auth Plugins

OAuth2/OIDC authentication plugins for [grepplabs/kafka-proxy](https://github.com/grepplabs/kafka-proxy), designed for integration with generic identity providers (Keycloak, Dex, etc.).

## Overview

This module provides **3 Go binaries** embedded in a Docker image based on `grepplabs/kafka-proxy:0.4.3-all`:

| Binary | Role | kafka-proxy mechanism |
|---|---|---|
| `oidc-token-verifier` | Validates incoming credentials (user/password or JWT) on the `kp-consumer` side | `--auth-local-command` (PLAIN) |
| `oidc-token-info` | Validates an incoming OAUTHBEARER token on the `kp-provider` side | `--auth-local-command` (OAUTHBEARER) |
| `oidc-token-provider` | Generates an OIDC token via client credentials flow | `--sasl-plugin-command` (OAUTHBEARER) |

## Authentication Flow Architecture

```
kcat/application
    │  SASL PLAIN (user:pass or user:JWT)
    ▼
kp-consumer  ◄── oidc-token-verifier (validates incoming credential)
    │              └─ static: user:pass map
    │              └─ JWT: JWKS validation
    │  SASL OAUTHBEARER  ◄── oidc-token-provider (generates OIDC token)
    ▼
kp-provider
    │  SASL OAUTHBEARER  ◄── oidc-token-info (validates received token)
    ▼
Kafka broker
    │  SASL OAUTHBEARER or PLAIN+TLS
    ▼
```

> **Dual-layer auth**: `kp-consumer` simultaneously handles *incoming* auth (verifier) and *outgoing* auth toward the provider (provider).

## Plugin Reference

### `oidc-token-verifier` — Incoming credential validation (PLAIN)

Used on the **consumer proxy** side to validate credentials received from Kafka clients (kcat, application).

Supports **two modes** in the same instance:
1. **Static users**: `user:password` provided via `--static-user`
2. **JWT OIDC**: the password is a Bearer token (`eyJ...`), validated via JWKS

**CLI flags:**
| Flag | Env fallback | Description |
|---|---|---|
| `--jwks-url` | `VERIFIER_JWKS_URL` | JWKS endpoint URL of the IDP |
| `--client-id` | `VERIFIER_CLIENT_ID` | Expected audience in the JWT token |
| `--allowed-issuer` | _(none)_ | Allowed issuers (comma-separated) |
| `--tenant-id` | _(none)_ | Tenant ID (Active Directory) — auto-configures JWKS URL if `--jwks-url` is empty |
| `--static-user` | `AUTH_STATIC_USERS` | Static users in `user:pass` format (repeatable) |
| `--debug` | `DEBUG=true` | Enables debug logging |

**Example invocation (by kafka-proxy-k8s-manager):**
```bash
/usr/local/bin/oidc-token-verifier \
  --debug \
  --client-id=my-kafka-client \
  --allowed-issuer=http://keycloak:8080/realms/master \
  --static-user=$(AUTH_STATIC_USERS)
```

> **Important**: `--static-user=$(AUTH_STATIC_USERS)` is resolved by the container shell, where `AUTH_STATIC_USERS` is an environment variable injected from a Kubernetes Secret in `user1:pass1,user2:pass2` format.

---

### `oidc-token-info` — Incoming OAUTHBEARER token validation

Used on the **provider proxy** side to validate OAUTHBEARER tokens received from the consumer proxy.

**CLI flags:**
| Flag | Env fallback | Description |
|---|---|---|
| `--jwks-url` | `INFO_JWKS_URL` | JWKS endpoint URL of the IDP |
| `--client-id` | `INFO_CLIENT_ID` | Expected audience in the JWT token |
| `--allowed-issuer` | `INFO_ALLOWED_ISSUERS` | Allowed issuers |
| `--required-scope` | `INFO_REQUIRED_SCOPE` | Required scope(s) in the token |
| `--debug` | `DEBUG=true` | Enables debug logging |

**Example invocation:**
```bash
/usr/local/bin/oidc-token-info \
  --debug \
  --client-id=kafka-proxy \
  --jwks-url=http://keycloak:8080/realms/master/protocol/openid-connect/certs \
  --allowed-issuer=http://keycloak:8080/realms/master
```

---

### `oidc-token-provider` — OAUTHBEARER token generation (Client Credentials)

Used on the **consumer proxy** side to obtain an OIDC token before connecting to the provider proxy.

**CLI flags:**
| Flag | Env fallback | Description |
|---|---|---|
| `--client-id` | `PROVIDER_CLIENT_ID` | OIDC Client ID |
| `--client-secret` | `PROVIDER_CLIENT_SECRET` | OIDC Client Secret |
| `--token-url` | `PROVIDER_TOKEN_URL` | IDP token endpoint URL |
| `--scope` | `PROVIDER_SCOPE` | Requested scope(s) |
| `--debug` | `DEBUG=true` | Enables debug logging |

The obtained token is **cached internally** until 10 seconds before its expiration. If the IDP does not return `expires_in`, the cache defaults to 1 hour.

**Example invocation:**
```bash
/usr/local/bin/oidc-token-provider \
  --client-id=kafka-proxy \
  --client-secret=MY_SECRET \
  --token-url=http://keycloak:8080/realms/master/protocol/openid-connect/token \
  --scope=openid
```

## Internal library `internal/oidc`

### `Validator` (`core.go`)
- Validates JWTs via JWKS (RSA only)
- Caches public keys in memory (TTL: 1 hour)
- Verifies: signature, expiration, audience (`AllowedAuds`), issuer (`AllowedIssuers`), scopes (`RequiredScopes`)
- Supports both `scope` (Keycloak standard) and `scp` (Azure AD) claims
- Multi-value parsing via comma or space (`ParseStringSlice`)

### `Provider` (`provider.go`)
- Implements the `client_credentials` flow via `http.PostForm`
- Caches the token until 10s before expiration (`expires_in`)
- Thread-safe via `sync.Mutex`

## Docker Image

The final image is built on `grepplabs/kafka-proxy:0.4.3-all` (multi-arch support: amd64/arm64).

```dockerfile
FROM grepplabs/kafka-proxy:0.4.3-all
COPY bin/oidc-token-provider  /usr/local/bin/
COPY bin/oidc-token-verifier  /usr/local/bin/
COPY bin/oidc-token-info      /usr/local/bin/
```

Binaries are placed in `/usr/local/bin/` and executable by `USER 1000` (non-root).

## Troubleshooting

### `main redeclared in this block`
A test file `patch_main.go` was accidentally left in a `cmd/` directory. Deleting it is enough to fix the build error.

### JWT validation fails
- Verify that `--jwks-url` points to the correct Keycloak realm
- Ensure `--client-id` matches the `audience` in the token (`aud` claim)
- Check that the token is not expired
- Enable `--debug` to see validation details in the pod logs

### Static users not working
- `AUTH_STATIC_USERS` is injected from a **Kubernetes Secret** — check its value with `kubectl get secret <name> -o yaml`
- The expected format for `--static-user` is `user:pass` (one per flag, or CSV via env var)
- In debug mode, loaded usernames are logged at startup

### Provider cannot obtain a token
- Check network connectivity to the IDP from the pod (`kubectl exec`)
- Ensure `PROVIDER_CLIENT_SECRET` is correctly mounted from the Secret
- Check Keycloak logs for `invalid_client_credentials` errors
